This page tells you how to get and use a Chromium-based browser (nicknamed Dartium) that includes the Dart virtual machine (VM). This browser can execute Dart web apps directly, so you don't have to compile your code to JavaScript until you're ready to test on other browsers.

Important: This browser is a technical preview, and it might have security and stability issues.
Do not use Dartium as your primary browser!

Downloading and installing the browser

If you have an up-to-date version of Dart Editor, you already have Dartium.

If you don't have Dart Editor or want a different version of Dartium, download the Dartium ZIP file for your platform:

  • Dartium for Windows
  • Dartium for Linux
  • Dartium for Mac

Note: The Dartium binary will expire after a few days. When that happens, you'll need to download a new copy if you want to continue using Dartium.

You don't usually need to do anything special to install Dartium: just unarchive the ZIP file. If you want Dart Editor to launch a particular copy of Dartium, then put that copy inside the dart-sdk directory of your Dart Editor installation directory, replacing the original copy of Chromium.

Launching the browser

To launch Dartium, navigate to its directory in your finder, and double-click the blue logo labeled Chromium. Or use Dart Editor or the command line.

If launching fails and you're using 64-bit Linux, you probably need to install 32-bit libraries. See bug #2052 for details and tips.

If you already use Chromium: If another version of Chromium is open, then you could have a profile conflict. To avoid this, you can open Dartium or Chromium from the command line with the --user-data-dir flag.

Filing bugs

If you find a bug in Dartium, create an issue in the Dart project and use the Dartium bug template:

File a bug against Dartium

Detecting Dart support

Your Dart code will work in Dartium without any additional JavaScript code. However, you have the option of using JavaScript code to test whether the current browser supports Dart. To do so, look for the function webkitStartDart on the window.navigator object. For example:

{% pretty_code dart 0 %} if (!navigator.webkitStartDart) { // No native Dart support. window.addEventListener("DOMContentLoaded", function (e) { // ...Fall back to compiled JS... } }, false); } {% endpretty_code %}

Launching from the command line

Because Dartium is based on Chromium, all Chromium flags should work. In some cases, you might want to specify Dart-specific flags so that you can tweak the embedded Dart VM’s behavior. For example, while developing your web app, you might want the VM to perform as many checks as possible. To achieve that, you can enable checked mode (the VM’s --enable_type_checks flag) and assertion checks (--enable_asserts flag).

On Linux, you can specify flags by starting Dartium as follows:

{% pretty_code dart 0 %} DART_FLAGS='--enable_type_checks --enable_asserts' path/chrome {% endpretty_code %}

On the Mac:

{% pretty_code dart 0 %} DART_FLAGS='--enable_type_checks --enable_asserts' \ path/Chromium.app/Contents/MacOS/Chromium {% endpretty_code %}

Or (also on the Mac):

{% pretty_code dart 0 %} DART_FLAG='--enable_type_checks --enable_asserts' \ open path/Chromium.app {% endpretty_code %}
Tip: You can see the command-line flags and executable path of your current Chromium-based browser by going to chrome://version.
