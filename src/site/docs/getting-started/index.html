Sample #1: Hello, World!

Below this paragraph, you can see some Dart code inside of an app called Dart Board. If you click Dart Board's [PLAY] button—go ahead, you know you want to—you'll see the expected output (Hello, World!) appear at the bottom.

main() { var name = 'World'; print('Hello, ${name}!'); }

What's important about this sample's Dart code:

Top-level main() method
Having a main() method with no surrounding class is a Dart Board convention. It lets Dart Board know where to start executing your code. In a web app, you'd use directives instead—little statements at the top of your .dart file. (Don't worry, you'll learn about directives in the next page.)
Variable declared with var
You must use either the var keyword or a type name whenever you create a variable.
Output with print()
The print() method is {PENDING: another Dart Board convention?} . In a web app you'd use the DOM or something like console.log() instead. {QUESTION: correct?}
String interpolation with ${expression}
Dart lets you embed expressions within string literals. If the expression is just a variable, as it is in this sample, then you can omit the curly braces ({}). Another alternative for building strings is the plus operator (+). For example, the following three statements are equivalent {PENDING: are they truly equivalent, or do they just result in the same string?} : 
print('Hello, ${name}!');
print('Hello, $name!');
print('Hello, ', + name + '!');

Sample #2: Hello, Class!

The next sample adds a class to the previous code. Classes are important in Dart. They're how you create objects, and along with interfaces they define all the object types in Dart programs. {PENDING: correct? should we point to a primer on object-oriented programming? if so, which one?}

class Greeter { var prefix = "Hello, "; Greeter() {} greet(name) { print('$prefix$name'); } } main() { var greeter = new Greeter(); greeter.greet('Class!'); }

What's important about the preceding code:

class statement
This sample defines a class, named Greeter, that has the default superclass: Object. In Dart, all classes descend from Object, either directly or indirectly. Use the extends keyword if you want to specify a non-Object superclass.
Instance variable
Each Greeter object has a variable named prefix that's initialized to "Hello, ". (As in JavaScript, Dart string literals can be delimited by either single or double quotes.) Just as for regular variables, creating instance variables requires using var or a type.

If you don't like the default value of prefix, you can set it to something else: 

greeter.prefix = "Howdy, ";

Constructor
{PENDING: the spec says there should be a default constructor, but SSR doesn't work if you delete the constructor. } You can create instances of a class by invoking new followed by a constructor for the class—in this case, new Greeter(). The Greeter class has a single, no-argument constructor that (implicitly) calls the superclass's no-argument constructor. The superclass of Greeter is Object, so the code new Greeter() invokes the Object() constructor.
Instance method
The greet() method defines a function that's tied to a Greeter object. For a one-line method such as greet(), you can use a special shorthand that features the => operator: 
// Equivalent of greet definition above:
greet(name) => print('$prefix$name');
Note: You currently get a warning about returning a value from a void function. Ignore it; the syntax is valid, and we expect the warning to disappear.

No specified types
Specifying types is optional in Dart. Types can help you write and maintain your code, but they don't change how Dart programs behave. Here's what the preceding sample looks like with types (as well as the other changes suggested above): class Greeter { String prefix = "Hello, "; Greeter() {} void greet(String name) => print('$prefix$name'); } void main() { Greeter greeter = new Greeter(); greeter.prefix = "Howdy, "; greeter.greet('types and other cool stuff.'); }
Note: You might have expected to see a public keyword in the preceding code, but there's no such thing in Dart. Every type {PENDING: ??} in Dart is public unless its name starts with an underscore ("_"). For example, Greeter is public, but _Greeter (by convention) would be private.

class App implements Runnable {
  App() {}

  void run(ReceivePort port) {
    Logger.println('Hey, this is App.run.');
  }
}

void main() {
  Isolate.start(new App(), function(SendPort port) {
    Logger.println('Hey, this is main.');
  });
}
