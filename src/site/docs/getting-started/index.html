Sample #2: Hello, Class!

The next sample adds a class to the previous code. Classes are important in Dart. They're how you create objects, and along with interfaces they define all the object types in Dart programs. {PENDING: correct? should we point to a primer on object-oriented programming? if so, which one?}

class Greeter { Greeter() {} printGreeting(greeting) { Logger.println(greeting); } } main() { var greeter = new Greeter(); greeter.printGreeting('Hello, Class!'); }

What's important about the preceding code:

class statement
This sample defines a class, named Greeter, that has the default superclass: Object. In Dart, all classes descend from Object, either directly or indirectly. Use the extends keyword if you want to specify a non-Object superclass.
Constructor for the class
{PENDING: the spec says there should be a default constructor, but SSR doesn't work if you delete the constructor. } You can create instances of a class by invoking new followed by a constructor for the class—in this case, new Greeter(). The Greeter class has a single, no-argument constructor. The constructor's empty body means that the constructor just calls super(). Because the superclass of Greeter is Object, the code new Greeter() invokes the Object() constructor.
Variable declared with var
You must use either the var keyword or a type name whenever you create a variable.
No specified types for variables and parameters
Specifying types is optional in Dart. We left types out of this sample because we're just playing right now, but here's what the program might look like with types (the new types are bold): 
class Greeter {
  Greeter() {}

  void printGreeting(String greeting) {
    Logger.println(greeting);
  }
}

void main() {
  Greeter greeter = new Greeter();
  greeter.printGreeting('Hello, Types!');
}

Note: You might have expected to see a public keyword in the preceding code, but there's no such thing in Dart. Every type {PENDING: ??} in Dart is public unless its name starts with an underscore ("_"). For example, Greeter is public, but _Greeter is private.

Sample #3: Hello, Interface!

class App implements Runnable {
  App() {}

  void run(ReceivePort port) {
    Logger.println('Hey, this is App.run.');
  }
}

void main() {
  Isolate.start(new App(), function(SendPort port) {
    Logger.println('Hey, this is main.');
  });
}
