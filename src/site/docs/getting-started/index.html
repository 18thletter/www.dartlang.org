Hello, World!

Below this paragraph, you can see some Dart code inside of an app called Dartboard. If you click Dartboard's Play button {PENDING: img goes here}—go ahead, you know you want to—you'll see the expected output (Hello, World!) appear at the bottom.

main() { var name = 'World'; print('Hello, ${name}!'); }

About the code

This example illustrates some basic Dart features and conventions:

Top-level main() function
Having a main() function with no surrounding class lets Dart know where to start executing your code.
Variable declared with var
Whenever you create a variable in a Dart program, you must use the var keyword, the final keyword, or a type name to declare that variable.
Output with print()
Dart's print() function sends text to a console.
String literals
You can use either single or double quotes to mark strings: 'World' and "World" are equally valid.
String interpolation with ${expression}
Dart lets you embed expressions within string literals. If the expression is just a variable, as it is in this example, then you can omit the curly braces ({}). Another alternative for building strings is the plus operator (+). For example, the following three statements are equivalent: 
print('Hello, ${name}!');
print('Hello, $name!');
print('Hello,' + name + '!');

Hello, Class!

The next example adds a class to the previous code. If you already know a class-based language, the syntax for creating and using classes probably looks familiar. If not, you might want to do a web search for object-oriented concepts, and read up before continuing. {PENDING: What's a better search? Or is there something we can link to directly?}

Classes are important in Dart. They're how you create objects. All the object types in Dart programs are either classes or interfaces. (You'll learn about Dart interfaces a little later.)

class Greeter { var prefix = 'Hello,'; Greeter() {} greet(name) { print('$prefix $name'); } } main() { var greeter = new Greeter(); greeter.greet('Class!'); }

About the code

The preceding code shows some basic features of Dart classes:

class statement
This example defines a class, named Greeter, that has the default superclass: Object. In Dart, all classes descend from Object, either directly or indirectly. Use the extends keyword if you want to specify a non-Object superclass.
Instance variable
Just as for regular variables, creating instance variables requires using var or a type. Each Greeter object has its own copy of a variable named prefix that's initialized to 'Hello,'.

You can set instance variables directly (for example, greeter.prefix = 'Hi,'), or through constructors, or through setter methods. {PENDING: point to where constructors and setters are discussed further. Mention getters?}

Constructor
{ PENDING: the spec says there should be a default constructor, but Dartboard doesn't work if you delete the constructor. } You can create instances of a class by invoking new followed by a constructor for the class—in this case, new Greeter(). The Greeter class has a single, no-argument constructor that (implicitly) calls the superclass's no-argument constructor. The superclass of Greeter is Object, so the code new Greeter() invokes the Object() constructor.

If you need more than one constructor for a class, then you can specify named constructors—for example, Greeter.withPrefix(). For example: 

//In the Greeter class:
Greeter.withPrefix(this.prefix);

//In code that creates Greeter objects:
var greeter = new Greeter.withPrefix('Howdy,');

In that code, this.prefix is a shortcut that assigns the parameter's value to the instance variable prefix.

Instance method
The greet() method defines a function that's tied to a Greeter object.
No specified types
Specifying types is optional in Dart. Types can help you write and maintain your code, but they don't change how Dart programs behave.

Hello, Interface! (DELETE?)

Interfaces—types that define which methods a class provides—are important in Dart. In fact, most of the Dart libraries are defined in terms of interfaces, although in practice you don't need to worry about this. Many library interfaces have a factory class— a class that creates objects that implement the interface. You don't need to know what the factory class is; you just use the interface's constructor, which defers to the factory class's implementation of the constructor.

For example, if your code says new DateTime.now(), the factory class for the DateTime interface creates an object representing the current time that implements all the methods in the DateTime interface. You can find documentation on DateTime and other APIs in the Dart Library Reference.

An example that might surprise you is the relationship between num, int, and double. You might expect int and double to be primitive types, but they aren't. Instead, they're interfaces that extend num. Having the num type makes it easy to deal with integer and floating-point numbers without having to explicitly convert from one to the other.
