Sample #1: Hello, World!

If you run the following program—go ahead, you know you want to—you'll see the expected output (Hello, World!) at the bottom of the DartBoard. {PENDING: Running here doesn't work yet. You can try this code out in http://www/~knorton/ssr.html.}

main() { Logger.println('Hello, World!'); }

What's important about this sample:

  • It has a main() method with no surrounding class.
    This is a DartBoard convention; it lets DartBoard know where to start executing your code.
  • It prints output using Logger.
    Again, this is a DartBoard convention. Server-side test code can also use Logger, but in a web app you'd use the DOM instead. {PENDING: check}

Sample #2: Hello, Class!

Let's add a class to the sample. Classes are important in Dart. They're how you create objects, and along with interfaces they define all the object types in Dart programs. {PENDING: point to a primer on object-oriented programming?}

main() { class Greeter { Greeter() {} printGreeting(greeting) { Logger.println(greeting); } } main() { var greeter = new Greeter(); greeter.printGreeting('Hello, Class!'); }

What's important about the preceding code:

  • It uses class to create a class named Greeter that inherits from the root class, Object.
  • Defines a constructor, Greeter()
