This page tells you how to find and use prerelease versions of the Dart SDK. The SDK includes the following tools:

  • Dart-to-JavaScript compiler (aka frog)
  • Dart VM
  • Dart libraries

For more information about the SDK, see the blog post Dart SDK Quick Tour.

Note: The SDK is in very early days, and it might not always work. If you just want to play with Dart, try Dartboard. For a real development environment, use Dart Editor.

Getting the SDK

If you have an up-to-date version of Dart Editor, you already have the SDK.

If you don't have Dart Editor or want a different version of the SDK, download the SDK ZIP file for your platform:

  • Windows SDK
  • Mac SDK
  • Linux SDK

Or live dangerously: download the latest, untested build of the SDK. These continuous builds are updated at least daily, and sometimes much more often.

  • Windows SDK (latest)
  • Mac SDK (latest)
  • Linux SDK (latest)

After you download and unzip the SDK, you'll have a directory named dart-sdk. Its bin directory has frogc (the Dart-to-JavaScript compiler) and dart (the Dart VM).

Using frog

To compile a Dart file to JavaScript, use bin/frogc. For example:

{% pretty_code console 0 %} $DART_SDK/bin/frogc test.dart {% endpretty_code %}

This command produces a .js file that contains the JavaScript equivalent of your Dart code.

Using the VM

To run a Dart file in the Dart VM, use bin/dart. For example:

{% pretty_code console 0 %} $DART_SDK/bin/dart test.dart {% endpretty_code %}

Print all the command-line options with --print-flags:

{% pc console 0 %} $DART_SDK/bin/dart --print-flags Test {% endpc %}

That third argument is required for now, until we fix bug #2837.

Enabling type checks

Dart programs run in one of two modes: checked or production. In checked mode, assignments are dynamically checked, and certain violations of the type system raise exceptions at run time. In production mode, static type annotations have no effect.

During development, we recommend using checked mode. You can run in checked mode with a command-line flag: {% pretty_code console 0 %} $DART_SDK/bin/dart --enable_type_checks test.dart {% endpretty_code %}

Enabling assertions

An assertion checks a boolean condition and, if false, raises an exception. The assert() statements do not run in production mode. We recommend that you enable assertions during development.

You can enable assertions with a command-line flag: {% pretty_code console 0 %} $DART_SDK/bin/dart --enable_asserts test.dart {% endpretty_code %}
