Dart Editor is a lightweight, open-source editor based on Eclipse components. You can use it to edit and run Dart apps, as well as to invoke the Dart-to-JavaScript compiler.

Updating Dart Editor

If you already have Dart Editor and want to keep its state, install the new Dart Editor on top of the old one. For example, put the new ZIP file beside the old one, and then use the command-line interface to unzip on top of the old files:

{% pc console 0 %} % cd dart-editor-installation-directory % unzip dart-editor-latest-file.zip replace dart/some-file? [y]es, [n]o, [A]ll, [N]one, [r]ename: A {% endpc %}

See the change log for a list of release dates and features.

Getting the continuous build

We recommend that you use the latest supported version of Dart Editor—the version linked to above. It's known as the integration build, and it's usually updated once per week.

Another option is to use the continuous build—the most recent, untested version of Dart Editor. To get it, download the 32-bit or 64-bit ZIP file for the Linux continuous build. ZIP file for the Mac continuous build. 32-bit or 64-bit ZIP file for the Windows continuous build.

Running apps

To run any Dart app, just click Dart Editor's Run button while any file in that app is selected. If you're working on a web app, Dart Editor brings up a browser window that displays the app's HTML page, with the app's code running inside it.

By default, Dart Editor launches both web apps and server apps in checked mode, with debugging enabled. When you execute a web application, by default Dart Editor launches Dartium.

You can specify another browser or mode using Tools > Manage Launches. If you specify a non-Dartium browser, Dart Editor automatically compiles the app to JavaScript.

Viewing samples

The Welcome page of Dart Editor displays a few samples. To open a sample and look at its source code, just click the sample's image.

If you don't see the Welcome page, you probably closed it. Get it back with Tools > Welcome Page.

Another way to open a sample is through the File menu. For example:

  1. Go to the File menu, and choose Open.... Or use the keyboard shortcut (Ctl+O or, on Mac, Option+O).

  2. Navigate to your Dart installation directory, then to samples/sunflower. Double-click Sunflower.dart.

Browsing APIs

With Dart Editor, you can easily find where APIs are declared, outline the contents of a Dart file, and auto-complete references to APIs. The following mini-tutorial assumes you're viewing the Sunflower sample in Dart Editor.

  1. Find where variables and types are declared.

    1. In the Editor view of Sunflower.dart, hold down the Control key (Command key on Mac) and move the mouse over the source code. A line appears under each item that's declared elsewhere in this .dart file or in another .dart file.

    2. Continuing to press Control or Command, hover over the string Math, (at the top of the Sunflower() constructor), and click.

      The editor displays the file that declares the Math type.

      Tip: You can also use this trick to find declarations within the same file.

  2. Click the left arrow button (to the left of the Run button ) to go back to Sunflower.dart.

  3. Either press Alt+O (Option+O on Mac) or right-click and choose Quick Outline.

    A panel comes up displaying the classes, methods, and fields declared in Sunflower.dart. It looks something like this:

    You can reduce the size of the list by typing one or more characters. For example, if you type x, only the xc variable appears.

    If you choose an item from the list—for example, xc— the editor scrolls to its declaration in Sunflower.dart.

  4. Now try Dart Editor's auto-complete feature.

    1. Find the declaration of TAU, near the end of Sunflower.dart:

      {% pretty_code dart 0 %} static final TAU = Math.PI * 2;{% endpretty_code %}

    2. Delete everything to the right of Math, and then type a period (.). After a short pause, a list of autocomplete suggestions comes up, looking something like this:

    3. Type or use the arrow keys to select PI, and then press Enter or Return to dismiss the panel.

    4. Complete your changes—for example, if you think sunflower seeds should be semi-circular, use Math.PI instead of Math.PI*2—and then save the file.

      If Dart Editor displays an error after you save, you've made a syntax error. Fix it or undo your typing, and then save again.

    5. Click the Run button to run your changed version.

    6. Back in Dart Editor, edit the Dart code again. This time use Ctl+Space to bring up the auto-complete suggestions.

      For example, type Mat, then Ctl+Space to see a list of classes and interfaces that start with "Mat".
