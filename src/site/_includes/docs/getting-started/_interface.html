Interfaces—types that define which methods a class provides—are important in Dart. In fact, most of the Dart libraries are defined in terms of interfaces, although you don't usually need to worry about this because many library interfaces have a factory class— a class that creates objects that implement the interface. You don't need to know anything about the factory class. You just use the interface's constructor, which defers to the factory class's implementation of the constructor.

For example, if your code says new DateTime.now(), the factory class for the DateTime interface creates an object representing the current time that implements all the methods in the DateTime interface. You can find documentation on DateTime and other APIs in the Dart Library Reference.

An example that might surprise you is the relationship between num, int, and double. You might expect int and double to be primitive types, but they aren't. Instead, they're interfaces that extend num. Having the num type makes it easy to deal with integer and floating-point numbers without having to explicitly convert from one to the other.

class Greeter implements Comparable { String prefix = 'Hello,'; Greeter() {} Greeter.withPrefix(this.prefix); greet(String name) => print('$prefix $name'); /* Implement the Comparable interface. */ /** Compares the prefixes of two Greeters. */ int compareTo(Greeter other) => prefix.compareTo(other.prefix); } void main() { Greeter greeter = new Greeter(); Greeter greeter2 = new Greeter.withPrefix('Hi,'); num result = greeter2.compareTo(greeter); if (result == 0) { greeter2.greet('you are the same.'); } else { greeter2.greet('you are different.'); } }
