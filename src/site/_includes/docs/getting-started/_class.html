Classes are important in Dart. They're how you create objects. All the object types in Dart programs are either classes or interfaces. (You'll learn about Dart interfaces a little later.)

class Greeter { var prefix = 'Hello,'; Greeter() {} greet(name) { print('$prefix $name'); } } main() { var greeter = new Greeter(); greeter.greet("Class!"); }

About the code

The preceding code shows some basic features of Dart classes:

class statement
This example defines a class, named Greeter, that has the default superclass: Object. In Dart, all classes descend from Object, either directly or indirectly. Use the extends keyword if you want to specify a non-Object superclass.
Instance variable
Just as for regular variables, creating instance variables requires using var, final, or a type. Each Greeter object has its own copy of a variable named prefix that's initialized to 'Hello,'.

You can set instance variables directly (for example, greeter.prefix = 'Hi,'), or through constructors, or through setter methods. {PENDING: Really should discuss setters further, either here or somewhere else that we point to.}

Constructor
{ PENDING: the spec says there should be a default constructor, but Dartboard doesn't work if you delete the constructor. } You can create instances of a class by invoking new followed by a constructor for the class—in this case, new Greeter(). The Greeter class has a single, no-argument constructor that (implicitly) calls the superclass's no-argument constructor. The superclass of Greeter is Object, so the code new Greeter() invokes the Object() constructor.

If you need more than one constructor for a class, then you can specify named constructors—for example, Greeter.withPrefix(). For example: 

//In the Greeter class:
Greeter.withPrefix(this.prefix);

//In code that creates Greeter objects:
var greeter = new Greeter.withPrefix('Howdy,');

In that code, this.prefix is a shortcut that assigns the parameter's value to the instance variable prefix.

Instance method
The greet() method defines a function that's tied to a Greeter object.
No specified types
Specifying types is optional in Dart. Types can help you write and maintain your code, but they don't change how Dart programs behave.

Try this

Set the value of the prefix instance variable.
For example, add this after line 12: 
greeter.prefix = 'Hola,';
Convert the greet() method definition to use =>.
For a one-line method such as greet(), you can use a special shorthand: 
greet(name) => print('$prefix $name');

This special syntax makes the method return the value of the expression to the right of =>.

Add a named constructor to Greeter, and call it.
For example: 
//In the Greeter class:
Greeter.withPrefix(this.prefix = '');

//In code that creates Greeter objects:
var greeter = new Greeter.withPrefix('Howdy,');
Add one or more types to the example.
Here's what the example might look like if you add some types, as well as a couple of other changes suggested in this section: 
class Greeter {
  String prefix = 'Hello,';

  Greeter() {}

  greet(String name) => print('$prefix $name');
}

void main() {
  Greeter greeter = new Greeter();
  greeter.prefix = 'Howdy,';
  greeter.greet('types and other cool stuff.');
}
Note: You might have expected to see a public keyword in the preceding code, but there's no such thing in Dart. Every type in Dart is public unless its name starts with an underscore ("_"). For example, Greeter is public, but _Greeter (by convention) would be private.
