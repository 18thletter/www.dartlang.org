    Step 5: Try some Dart Editor features

    This section introduces you to some of Dart Editor's features, such as finding where APIs are declared, outlining the contents of a Dart file, and auto-completing references to APIs. We expect to add many more features over time.

    1. Learn how to find where variables and types are declared.

      1. In the Editor view of Sunflower.dart, hold down the Control key (Command key on Mac) and move the mouse over the source code. A line appears under each item that's declared elsewhere in this .dart file or in another .dart file.

      2. Continuing to press Control or Command, hover over the string Math, (at the top of the Sunflower() constructor), and click.

        The editor displays the file that declares the Math type.

        Tip: You can also use this trick to find declarations within the same file.

    2. Click the left arrow button (to the left of the Run button ) to go back to Sunflower.dart.

    3. Either press Alt+O (Option+O on Mac) or right-click and choose Quick Outline.

      A panel comes up displaying the classes, methods, and fields declared in Sunflower.dart. It looks something like this:

      You can reduce the size of the list by typing one or more characters. For example, if you type x, only the xc variable appears.

      If you choose an item from the list—for example, xc— the editor scrolls to its declaration in Sunflower.dart.

    4. Now try Dart Editor's auto-complete feature.

      1. Find the declaration of TAU, near the end of Sunflower.dart:

        {% pretty_code dart 0 %} static final TAU = Math.PI * 2;{% endpretty_code %}

      2. Delete everything to the right of Math, and then type a period (.). After a short pause, a list of autocomplete suggestions comes up, looking something like this:

      3. Type or use the arrow keys to select PI, and then press Enter or Return to dismiss the panel.

      4. Complete your changes—for example, if you think sunflower seeds should be semi-circular, use Math.PI instead of Math.PI*2—and then save the file.

        If Dart Editor displays an error after you save, you've made a syntax error. Fix it or undo your typing, and then save again.

      5. Click the Run button to run your changed version.

      6. Back in Dart Editor, edit the Dart code again. This time use Ctl+Space to bring up the auto-complete suggestions.

        For example, type Mat, then Ctl+Space to see a list of classes and interfaces that start with "Mat".

    Step 6: Create a new web app

    Now that you're familiar with Dart Editor, let's create a web app from scratch.

    1. Click the New Application button (at the upper left of Dart Editor) to create a new app. Alternatively, choose File > New Application... from the Dart Editor menu.

      A dialog comes up, looking something like this:

    2. Type in a name for your application—for example, HelloWorld. If you don't like the default directory, type in a new location or browse to choose the location. Make sure the application type is set to Web. Then click Finish to create the initial files for the app.

      A default Dart file appears in the Edit view, and a directory with a few files appears in the Files view. Your Dart Editor window should look something like this:

    3. Click the Run button to run your new app.

    4. View the HTML file in Dart Editor by double-clicking it in the Files view.

      The HTML file has one <script> tag with the type "application/dart", plus a second <script> tag that refers to a JavaScript file. That second tag lets non-Dart browsers find the JavaScript version of the Dart file.

    5. Modify the default Dart and HTML files as you like, using auto-completion, samples, and API reference documentation to figure out which APIs you can use.
