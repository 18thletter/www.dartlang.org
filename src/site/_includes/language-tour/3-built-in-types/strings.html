A Dart string is a sequence of Unicode character codes. You can use either single or double quotes to create a string:

{% pretty_code dart 0 %} var s1 = 'Single quotes work well for string literals.'; var s2 = "Double quotes work just as well."; var s3 = 'It\'s easy to escape the string delimiter.'; var s4 = "It's even easier to just use the other string delimiter."; {% endpretty_code %}

You can put the value of an expression inside a string by using ${expression}. If the expression is a variable, you can skip the {}.

{% pretty_code dart 0 %} var s = 'string interpolation'; print('Dart has $s, which is very handy.'); print('That deserves all caps. ${s.toUpperCase()} is very handy!'); {% endpretty_code %}

You can concatenate strings using adjacent string literals:

{% pretty_code dart 0 %} var s = 'string ''concatenation' " works even over line breaks "; print(s); // string concatenation works even over line breaks {% endpretty_code %}

Another way to create a multi-line string: use a triple quote with either single or double quotation marks.

{% pretty_code dart 0 %} var s1 = ''' You can create multi-line strings like this one. '''; var s2 = """This is also a multi-line string."""; {% endpretty_code %} {% comment %} In a multi-line string, the initial newline is NOT ignored (in Dartboard) (and neither is the last newline). {% endcomment %}

You can create a "raw" string by prefixing it with @.

{% pretty_code dart 0 %} var s = @'In a raw string, even \n is ignored.'; {% endpretty_code %} {% comment %} In a raw string, \' isn't ignored (in Dartboard). {% endcomment %}

As for all objects, you can check whether two strings are equivalent (have the same characters) using the == operator:

{% pretty_code dart 0 %} var name = 'NAME'; var greeting = "Hello, $name!"; var greetingTemplate = 'Hello, NAME!'; print(greeting == greetingTemplate); // true; they have the same characters {% endpretty_code %}

String methods

Each string literal has the type String. String has some handy methods, including some that let you determine whether a string starts with, ends with, or contains another string.

{% pretty_code dart 0 %} var fullName = 'Cuthbert Musgrave Girdlestone, III'; fullName.startsWith('Cuthbert'); // true fullName.endsWith('III'); // true fullName.contains(new RegExp('Musgrave')); // true {% endpretty_code %}

Strings are immutable objects, which means you can create them but you can't change them. If you look closely at the String API docs, you'll notice that none of the methods actually changes the state of a String. For example, the method replaceAll() returns a new String without changing the original String.

{% pretty_code dart 0 %} var greetingTemplate = 'Hello, NAME!'; var greeting = greetingTemplate.replaceAll(new RegExp("NAME"), 'Bob'); print(greeting == greetingTemplate); // false; greetingTemplate didn't change {% endpretty_code %} Learn more about string methods in the library tour.

StringBuffer methods

To programmatically generate a string, you can use StringBuffer. A StringBuffer doesn't generate a new String object until toString() is called.

{% pretty_code dart 0 %} var sb = new StringBuffer(); sb.add("Use a StringBuffer"); sb.addAll(["for ", "efficient ", "string ", "creation "]); sb.add("if you are ").add("building lots of strings."); var fullString = sb.toString(); print(fullString); // use a StringBuffer for efficient string creation // if you are building lots of strings sb.clear(); // all gone! {% endpretty_code %}
Note: StringBuffers are currently slow when compiled to JavaScript. See bug #1216 for details.
